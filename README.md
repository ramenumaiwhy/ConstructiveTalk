# ConstructiveTalk (建設的会話を促すアプリ)

創造的な瞬間を捉え、評価懸念を軽減することに焦点を当てた、建設的な会話を促進・記録するためのプログレッシブWebアプリケーションです。

## 🎯 MVP（最小実行製品）の範囲

### 基本機能
1. **LLMとの対話機能**
   - ユーザーがLLMと対話し、創造的なアイデアを引き出す
   - LLMがプロンプトを生成し、会話を促進

2. **コンテキスト記録**
   - アルコールレベル（0-5段階）
   - ムード（選択式）
   - 場所（自由入力）

3. **会話ログ管理**
   - Markdown形式での保存
   - 重要なポイントのマークとタグ付け

## 🌟 主な機能

- LLMとの対話を通じたアイデア創出
- コンテキスト情報のトラッキング
- Markdown形式での会話ログ保存
- 会話内容の構造化分析
- ムードやコンテキスト情報のトラッキング
- 使い慣れたインターフェースのLINE Bot連携
- Markdown形式でのエクスポート機能
- 会話内容の構造化分析
- GitHubと連携したデータバックアップ

## 🛠 技術スタック

- **フロントエンド**: Next.js + TypeScript + TailwindCSS
- **バックエンド**: Vercel + Supabase
- **Botプラットフォーム**: LINE Messaging API
- **データストレージ**: Markdown形式（GitHub連携）

## 📋 必要要件

- Node.js（最新のLTSバージョン推奨）
- LINE Developersアカウント
- Vercelアカウント
- Supabaseアカウント
- GitHubアカウント

## 🚀 はじめ方

1. リポジトリのクローン
```bash
git clone https://github.com/yourusername/ConstructiveTalk.git
cd ConstructiveTalk
```

2. 依存パッケージのインストール
```bash
npm install
```

3. 環境変数の設定
```bash
cp .env.example .env.local
```
`.env.local`に以下の設定を追加：
- LINE Channel Secret
- LINE Channel Access Token
- Supabase URL
- Supabase Anonymous Key

4. 開発サーバーの起動
```bash
npm run dev
```

## 🔧 設定方法

### LINE Botの設定
1. LINE Developersアカウントの作成
2. 新規チャンネルの作成（Messaging API）
3. WebhookのURL設定
4. 必要なLINE Bot機能の有効化

### Vercelへのデプロイ
1. GitHubリポジトリのインポート
2. 環境変数の設定
3. アプリケーションのデプロイ

### Supabaseの設定
1. 新規Supabaseプロジェクトの作成
2. データベーステーブルの設定
3. 認証設定の構成

## 📝 使用方法

1. LINE Botで会話を開始
2. コンテキスト情報（ムード、設定など）を追加
3. 創造的な議論を展開
4. 会話ログをMarkdown形式でエクスポート
5. 会話内容の振り返りと分析
